# Vox — Hands-Free Voice-to-Text Utility 🎙️⌨️

**Vox** — це глобальна фонова hands-free утиліта для введення тексту та команд голосом у будь-яке активне вікно системи (термінал, редактор коду, месенджер тощо). 
Вона оптимізована для розробників, які хочуть працювати без рук, та повністю підтримує українську мову.

Проєкт використовує клієнт-серверну архітектуру:
*   **Сервер (Backend):** Працює в Docker всередині Proxmox LXC контейнера. Використовує надшвидкий `faster-whisper` на CPU (AVX2).
*   **Клієнт (Desktop Client):** Фоновий демон на Python з системним треєм, красивим напівпрозорим оверлеєм та підтримкою симуляції введення в Windows та Linux (зокрема Wayland/X11).

## 🛠️ Крок 1: Налаштування та запуск Сервера на Proxmox LXC

### 1.1. Створення LXC контейнера в Proxmox 9.x
1. Створіть новий LXC контейнер у веб-інтерфейсі Proxmox на базі образу **Debian 12** або **Ubuntu 22.04 LTS**.
2. Виділіть для контейнера ресурси, наприклад:
   * **CPU:** 2 або 3 ядра.
   * **RAM:** 1 ГБ або 1.5 ГБ.
   * **Диск:** 8–10 ГБ.
3. **КРИТИЧНО НАЛАШТУВАННЯ:**
   Після створення контейнера, перейдіть у веб-інтерфейсі: 
   `Ваш LXC Контейнер -> Options -> Features -> Edit` 
   Поставте галочки біля:
   * **[ ✓ ] Nesting** (дозволяє запускати Docker всередині контейнера).
   * **[ ✓ ] Keyctl** (необхідно для авторизації та роботи Docker демона).

*Або через SSH консоль самого Proxmox хоста (файл `/etc/pve/lxc/<VMID>.conf`):*
```ini
features: keyctl=1,nesting=1
```

### 1.2. Встановлення Docker всередині LXC контейнера
Зайдіть у консоль вашого створеного LXC контейнера та виконайте команди:
```bash
# Оновлюємо репозиторії
apt update && apt upgrade -y

# Встановлюємо залежності для Docker
apt install -y curl ca-certificates gnupg

# Додаємо офіційний ключ GPG Docker
install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/debian/gpg | gpg --dearmor -o /etc/apt/keyrings/docker.gpg
chmod a+r /etc/apt/keyrings/docker.gpg

# Додаємо репозиторій Docker
echo \
  "deb [arch="$(dpkg --print-architecture)" signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/debian \
  "$(. /etc/os-release && echo "$VERSION_CODENAME")" stable" | \
  tee /etc/apt/sources.list.d/docker.list > /dev/null

# Встановлюємо Docker та Docker Compose
apt update
apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

### 1.3. Запуск сервера Vox
Скопіюйте файли проєкту на сервер та запустіть контейнер:
```bash
# Перейдіть у папку проєкту (де знаходиться docker-compose.yml)
cd /home/carnat/vox

# Запустіть Docker Compose в фоновому режимі
docker compose up -d
```
Сервер почне роботу на порту `8002`. Під час першого підключення клієнта сервер автоматично завантажить модель Whisper `base` (~140MB) та збереже її у кеш-папку `./models` на хості.

## 💻 Крок 2: Налаштування та запуск Клієнта (На вашому ПК)

Клієнтська частина запускається локально на робочому комп'ютері (підтримує Windows та Linux).

### 2.1. Встановлення залежностей
Перейдіть у папку клієнта та створіть віртуальне середовище Python:
```bash
cd /home/carnat/vox/client

# Створюємо віртуальне оточення
python3 -m venv venv

# Активуємо його
source venv/bin/activate

# Встановлюємо залежності
pip install -r requirements.txt
```

### 2.2. Завантаження локальної моделі Vosk для української мови
Для офлайн-активації голосом («привіт вокс») потрібна легка українська модель Vosk (~40 МБ):
1. Створіть папку `models` всередині папки `client`:
   ```bash
   mkdir -p models
   ```
2. Завантажте модель за посиланням: [vosk-model-small-uk-0.22.zip](https://alphacephei.com/vosk/models/vosk-model-small-uk-0.22.zip)
3. Розархівуйте її в папку `models/` так, щоб кінцевий шлях до моделі виглядав як:
   `models/vosk-model-small-uk-0.22/`

### 2.3. Налаштування прав для Wayland інжекції (Тільки для Linux Wayland)
Якщо ви використовуєте Linux з Wayland і хочете вводити текст через симуляцію клавіатури на рівні ядра (`uinput`), налаштуйте права доступу:
1. Додайте вашого користувача до групи `input`:
   ```bash
   sudo usermod -aG input $USER
   ```
2. Створіть udev-правило для надання прав запису в `/dev/uinput` вашій групі:
   ```bash
   echo 'KERNEL=="uinput", GROUP="input", MODE="0660", OPTIONS+="static_node=uinput"' | sudo tee /etc/udev/rules.d/99-uinput.rules
   ```
3. Перезапустіть систему або udev-демон, щоб правила набули чинності:
   ```bash
   sudo udevadm control --reload-rules && sudo udevadm trigger
   ```
*Примітка: Якщо ви не налаштуєте права, Vox автоматично переключиться на кросплатформний **Clipboard Fallback** метод (копіювання тексту та швидке натискання `Shift+Insert` з відновленням буфера), який працює без додаткових прав.*

### 2.4. Конфігурація клієнта
Відкрийте файл `config.py` та вкажіть IP-адресу вашого Proxmox LXC контейнера, де працює сервер:
```python
SERVER_WS_URL = "ws://<IP_АДРЕСА_LXC_КОНТЕЙНЕРА>:8002/stream"
WAKE_WORD = "привіт вокс"  # Активаційна фраза
```

### 2.5. Запуск клієнта
Переконайтеся, що ви знаходитесь у кореневій директорії проекту та віртуальне середовище активоване:
```bash
# Активуйте venv (якщо ще не активовано)
source venv/bin/activate

# Запустіть додаток
python3 vox_client.py
```

## 🕹️ Як користуватися Vox

1. **Запуск у фоні:** Після запуску в системному треї (Tray) з'явиться кольорова іконка-індикатор:
   * **Сірий колір:** Голосову активацію вимкнено в меню трею. Працює тільки активація кнопкою.
   * **Зелений/Синій колір:** Голосова активація активна. Клієнт локально прослуховує мікрофон на наявність фрази *"привіт вокс"*.
   * **Червоний колір:** Йде запис промови та стрімінг на сервер.
2. **Активація запису:**
   * **Голосом:** Вимовіть чітко **«привіт вокс»** (або вашу кастомну фразу).
   * **Клавіатурою:** Натисніть гарячу клавішу **`Ctrl + Space`** (працює завжди).
3. **Диктовка тексту:**
   * З'явиться красиве напівпрозоре вікно оверлея внизу екрану.
   * Диктуйте текст чи команди українською мовою. Оверлей показуватиме хід розпізнавання наживо.
   * Щоб завершити диктовку — просто замовкніть на **1.5 секунди**. 
   * Vox зафіксує тишу, введе розпізнаний текст прямо у поточний активний курсор (термінал чи редактор), автоматично натисне **`Enter`** (Hands-free Auto-Enter) та сховає оверлей.
   * *Альтернативно:* Ви можете зупинити запис повторним натисканням гарячої клавіші `Ctrl + Space`.
4. **Керування через трей:**
   * Клацніть правою кнопкою миші на іконку в треї, щоб увімкнути/вимкнути розпізнавання голосу чи повністю вийти з додатку.
